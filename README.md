# STEM Fair Stock Market Simulator

## Project Overview

An interactive, browser-based stock market simulation designed for educational presentations at STEM fairs. This application teaches upper elementary to middle school students (ages 10-14) about the stock market through a live simulation and narrative presentation.

### Target Audience
- Upper elementary to middle school students (ages 10-14)
- Group presentations, 10-15 minutes each
- Groups of 2-4 students per session

### Key Features
- **Real-time candlestick chart** showing stock price movements with OHLC (Open, High, Low, Close) data
- **Live price updates** with realistic market fluctuations every frame
- **Group portfolio tracking** showing each group's balance, shares owned, and transaction history
- **Event-triggered price changes** controlled by the presenter for educational scenarios
- **Simple, large UI** designed for young students (buy/sell buttons, clear information)

---

## Presentation Structure (Act 1, 2, 3)

### Act 1 - Company Origin
- Introduce a fictional company
- Explain growth trajectory
- Teach what "going public" means (IPO)

### Act 2 - Market Basics
- Explain stocks, shares, buying/selling concepts
- Introduce candlestick charts and how to read them
- Demonstrate how prices move based on supply and demand

### Act 3 - Interactive Events
- Present 4-6 real-world scenarios (product launches, scandals, competitor actions)
- Students decide to buy, sell, or hold shares based on news
- Show cause-and-effect between events and price movements

### File Structure

```

│   │   ├── chart.css                  # Styles specific to chart view
│   │   └── portfolio.css              # Styles specific to portfolio view
│   └── js/
│       ├── types.ts                   # TypeScript type definitions and interfaces
│       ├── state.ts                   # Centralized state management (data layer)
│       ├── storage.ts                 # Data access abstraction layer (localStorage → can be swapped for API)
│       ├── priceGenerator.ts          # Random price generation logic
│       ├── chart.ts                   # Canvas rendering for candlesticks
│       ├── main.ts                    # Initialization and event loop for chart view
│       ├── portfolio-main.ts          # Initialization and event loop for portfolio view
│       └── utils.ts                   # Helper/utility functions
├── js/                                # Compiled JavaScript output (generated by TypeScript)
├── assets/                            # Images, fonts, etc. (future use)
└── node_modules/                      # Project dependencies (generated by npm)
```

**Source files** (in `src/`) are written in **TypeScript** and compiled to **JavaScript** (in `js/`). The HTML files reference the compiled JavaScript.

### Data Model

The application tracks the following data:

#### Market Data
- **Current stock price** (updates every frame as market fluctuates)
- **Candlestick history** (array of completed candles with OHLC data)
- **Current open candle** (the candle being built in real-time, updating each frame)
- **Market events** (triggered by presenter to simulate real-world scenarios)

#### Group Portfolio
- **Group name/ID**
- **Current balance** (remaining cash)
- **Shares owned** (quantity of shares held)
- **Transaction history** (log of all buys/sells with timestamps and prices)
- **Portfolio value** (current balance + (shares × current price))

## Architecture Principles

### 1. **Separation of Concerns**
- **state.ts**: Manages all application state (data)
- **storage.ts**: Handles getting/setting data (abstraction layer)
- **Views** (index.html, portfolio.html): Display data and handle user interactions
- **Business logic**: Price generation, chart rendering in separate modules
- **types.ts**: Centralized TypeScript interfaces and type definitions

### 2. **Data Abstraction Layer (storage.ts)**
All data access goes through `storage.ts` functions. Currently uses `localStorage`, but this design allows easy swapping to API calls:
- `getState()` - retrieve current state
- `setState(data)` - update state
- `updatePrice(price)` - update current price
- `buyShares(groupId, quantity, price)` - execute buy transaction
- `sellShares(groupId, quantity, price)` - execute sell transaction
- `triggerEvent(eventData)` - trigger market event

**Future migration**: To move to a server-based system, only `storage.js` needs to change. All other code remains the same.

### 3. **Multi-View Synchronization**
Both `index.html` and `portfolio.html` import the same state and storage modules:
- Changes on one view are immediately reflected on the other
- Uses `localStorage` events or polling to keep views in sync
- Presenter controls events from their laptop

### 4. **Rendering Loop**
- **main.js**: Runs continuous animation loop for chart (60 FPS)
- Each frame: fetches current state, redraws canvas with updated candle data
- Price updates happen at consistent intervals (e.g., every 10-30 frames)

## Data Flow

```
Presenter Action / Auto Update
        ↓
   priceGenerator.ts (generates new price)
        ↓
   state.ts (updates current price, candle data)
        ↓
   storage.ts (persists to localStorage)
        ↓
   index.html & portfolio.html (both read updated state)
        ↓
   chart.ts / portfolio.ts (render new data to screen)
```

## Getting Started

### Prerequisites
- Node.js (v14 or higher)
- npm (comes with Node.js)

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/yourusername/stock-market-simulator.git
   cd stock-market-simulator
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Compile TypeScript to JavaScript**
   ```bash
   npm run build
   ```

   Or, to watch for changes during development:
   ```bash
   npm run watch
   ```

4. **Open in browser**
   - For the market chart view: Open `src/index.html` in your browser
   - For the portfolio view: Open `src/portfolio.html` in your browser
   - Or serve with a local server:
     ```bash
     npx http-server src/
     ```

### Build Output
TypeScript files in `src/js/` are compiled to JavaScript in `js/`. The HTML files reference the compiled JavaScript, so after making changes to `.ts` files, you must run `npm run build` (or use `npm run watch` for automatic compilation).

## Key Concepts

### Candlestick Chart
- **Open**: Price at start of time period
- **High**: Highest price during period
- **Low**: Lowest price during period
- **Close**: Price at end of period
- Candles update in real-time until they "close" (every 30 seconds)
- A new candle then begins

### Group Portfolio
- Each group gets $10,000 starting capital
- Groups make buy/sell decisions during pauses
- System tracks portfolio value and transaction history
- End of session shows performance ranking

---

## Deployment & Usage

### Local Setup
1. Both monitors/windows open on presenter's laptop
2. `index.html` displayed in full screen on Monitor 1 (market chart)
3. `portfolio.html` displayed in full screen on Monitor 2 (student trading interface)
4. Presenter controls events via keyboard shortcuts or hidden UI controls

### Starting a New Session
- Reset button clears all portfolio and transaction data
- Chart starts from initial price
- New groups are created with fresh $10,000 balance

### No Server Required
- Everything is client-side using `localStorage`
- Works offline
- Quick to reset between sessions
